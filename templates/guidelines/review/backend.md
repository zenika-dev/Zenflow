# Backend Review Protocol

Purpose: define the review scope and process for backend changes. All audit criteria come from the architecture guideline.

## Scope

Apply this protocol when reviewing backend handovers or backend files.

## How To Review

Audit every changed file against the rules in the architecture guideline (`@.github/guidelines/architecture-backend.md`). Use every section of the guideline as a checklist — including project structure, API conventions, persistence rules, and testing strategy.

Do not invent criteria beyond what the architecture guideline defines.
