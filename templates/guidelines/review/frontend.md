# Frontend Review Protocol

Purpose: define the review scope and process for frontend changes. All audit criteria come from the architecture guideline.

## Scope

Apply this protocol when reviewing frontend handovers or frontend files.

## How To Review

Audit every changed file against the rules in the architecture guideline (`@.github/guidelines/architecture-frontend.md`). Use every section of the guideline as a checklist — including frontend conventions, accessibility rules, styling token rules, and testing strategy.

Do not invent criteria beyond what the architecture guideline defines.
